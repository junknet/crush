Resolve the call hierarchy around a Nim symbol via the language server (`textDocument/prepareCallHierarchy` + `callHierarchy/incomingCalls` / `outgoingCalls`).

Use this when you need to know:

- **Who calls this proc?** (`direction: "incoming"`) — every caller, across files.
- **What does this proc call?** (`direction: "outgoing"`) — every callee, including templates and macros invoked from the body.
- Both at once (`direction: "both"`, default).

AGENT_GUIDE §10 notes the nimlsp call detection covers ~80% of normal proc/template direct calls; uncovered shapes include infix-operator calls, multi-method dispatch ambiguity, and bodies generated by macro/template expansion. Treat the result as authoritative for direct calls, not exhaustive for indirect ones.

Input:
- `file_path`: absolute or relative path to a Nim source file.
- `line`: 1-based line of the cursor on the symbol.
- `character`: 1-based column of the cursor on the symbol.
- `direction`: one of `"incoming"`, `"outgoing"`, or `"both"` (default `"both"`).

Output: per resolved hierarchy item, a section listing incoming and/or outgoing edges with `name @ path:line:col` for each. Empty result means the cursor is not on a callable symbol.
